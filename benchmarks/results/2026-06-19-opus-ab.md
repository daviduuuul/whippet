# A/B run — 2026-06-19, Claude Opus 4.8

First real paired A/B on the fixture harness. Replaces the v1 self-reported seed
(`opus-4.8-2026-06.json`, removed) with objectively scored runs. Raw observations:
`runs.jsonl` (75 lines).

## Method

**Model-instruction A/B** (not a full Claude-Code-session A/B): one model
(`claude-opus-4-8`), three arms that differ **only** by the discipline prepended
to an otherwise identical prompt:

- **off** — solve it as a competent dev normally would; explicitly ignore any lean/whippet discipline.
- **baseline** — the one-line nudge from METHODOLOGY ("minimal code that works; reuse existing/stdlib/native before adding code or deps; report tersely").
- **whippet** — the full ladder + never-cut-validation/security + terse reporting.

5 fixtures × 3 arms × **5 runs** = 75 candidates. Candidates were generated by
subagents (blind to the hidden graders), then scored **deterministically** by
`scripts/bench-score.js` — the numbers do not depend on anything the generating
agent reported. Pre-registered metrics: correctness (hard gate), `loc_added`,
`files_added`, `deps_added`, `reused`, reported per category.

## Findings (honest)

1. **Correctness parity — 25/25 for every arm, every category**, including
   `trap_overcut` (validation): even the *off* arm kept the input guard. On a
   strong 2026 model, correctness on these tasks is universal; whippet does not
   improve it because the model is already correct.
2. **Diff size is the only discriminator.** Mean LOC added: whippet **3.9**,
   baseline **3.6**, off **19.2** — the disciplined arms produce ~5× smaller
   diffs than an un-nudged agent, at equal correctness. Per category, *off* is
   3–10× larger everywhere (e.g. `trap_yagni` 0.4/1.8 vs 17.6; `trap_overcut`
   14/10.6 vs 47.2 — validation legitimately needs code, so the lean arms keep
   it *and* stay smaller).
3. **whippet ≈ baseline.** The full plugin does **not** beat the one-line nudge
   on these fixtures (3.9 vs 3.6 LOC; both 25/25 correct; deps 0/0). Per
   METHODOLOGY's own rule, that null is published, not hidden: on small,
   single-function tasks the elaboration buys nothing the one-liner doesn't.
4. **No arm added a dependency** (0/0/0). Reuse of the flagged helper: whippet
   15/15, off 15/15, baseline 13/15.

## Caveats (what this does NOT show)

- Small **single-function** fixtures, **n=5** per cell → wide Wilson CIs; the
  correctness ceiling (25/25) means LOC is the only axis with spread.
- **Model-instruction** A/B, not full Claude-Code sessions (no tool-use, no
  multi-turn, no session hook). It isolates the *discipline-as-prompt* effect.
- One model, one day. Temperature-0 is not deterministic; re-runs will vary.
- The reviewable-diff gap (whippet/baseline ≪ off) is consistent with DORA p.58's
  small-batch lever but measured on toy tasks, not delivery instability.

A bigger, multi-step fixture set and a full-session arm are the next step before
any headline claim rests on this.
